# 任务路线编辑

任务路线编辑页用于把一次模板执行轨迹整理成后续可复用的路线。你可以把它理解为“从刚才那次执行记录里，挑出以后还需要重复走的步骤”。

## 什么时候进入路线编辑？

任务路线归任务模板所有。建议在模板至少跑过一次后再进入路线编辑。即使模板最后失败，只要中间产生了有效操作记录，也可以打开路线编辑页查看并整理。

入口和前端一致：

- 任务模板列表或模板详情页进入某个模板。
- 在模板详情页点击 **打开任务路线编辑页**。
- 工作流步骤引用模板时，可以先打开对应模板详情页，再进入路线编辑页。

!!! note "需要先保存模板"
    新建模板时不会显示路线编辑入口。前端会提示“任务配置至少保存一次后，才能编辑对应的任务路线。”

## 页面区域

| 前端区域 / 按钮 | 说明 |
| --- | --- |
| 任务路线目标 | 显示当前编辑的是哪个模板对应的路线。 |
| 已保存路线 | 显示当前已经保存过的路线。如果还没有保存，会显示暂无已保存路线。 |
| 当前路线 | 显示最近一次捕获到、准备保存进路线的动作。 |
| 移出路线 | 把某个动作从当前路线中移出。被移出的动作不会保存进最终路线。 |
| 已移出路线 | 显示被移出的动作。保存前可以恢复。 |
| 恢复到路线 | 把已移出的动作恢复回当前路线。 |
| 手动保存路线 | 把当前保留的动作保存成该模板的正式路线。 |
| 回放结束后直接结束任务 | 路线回放成功后是否直接结束当前模板运行。这个开关只有点击“手动保存路线”时才会一起保存。 |

## 基本流程

1. 先运行一次模板，或运行包含该模板的工作流。
2. 打开该模板的路线编辑页。
3. 查看 **当前路线** 中最近一次捕获到的操作轨迹。
4. 用 **移出路线** 去掉启动页等待、误触、无关弹窗处理等动作。
5. 如果移错了，在 **已移出路线** 里点击 **恢复到路线**。
6. 点击 **手动保存路线**。
7. 再运行一次模板或工作流，确认路线能正常回放。

## 哪些动作应该删除？

建议删除：

- 误点、重复点击、无效等待。
- 与任务目标无关的弹窗处理。
- 只在某一次运行中偶然出现的页面。
- 进入错误页面后又返回的修正动作。

建议保留：

- 打开目标 App 后的固定入口。
- 进入固定 tab、固定页面的点击。
- 到达任务关键页面前的稳定跳转步骤。

## 回放后直接结束任务

“回放结束后直接结束任务”用于控制路线回放成功后的行为：

- 开启：路线回放成功后，当前模板运行直接结束。
- 关闭：路线回放成功后，继续进入后续视觉执行。

这个选项和路线一起保存。修改开关后，需要点击 **手动保存路线** 才会生效。

!!! warning "不要误开"
    如果你的路线只负责“进入目标页面”，后面还需要模型继续填写、判断或提交，就不要开启“回放结束后直接结束任务”。只有当路线本身已经完整完成模板目标时，才适合开启。

## 保存后的验证

保存路线后建议立刻再跑一次，观察：

- 是否更快到达目标页面。
- 是否跳过了不必要的模型判断。
- 是否出现路线回放失败。
- 失败后是否能正常降级到视觉执行。

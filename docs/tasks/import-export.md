# 便携包导入与导出

便携包用于在设备之间迁移任务模板、工作流和模板路线。它会尽量保留任务说明、目标应用、用户操作文档、步骤编排和已保存路线，让另一台设备导入后再做本机适配。

## 什么时候需要导出？

适合导出：

- 你在一台设备上调好了一个任务模板，想放到另一台设备使用。
- 你想备份一个已经编排好的工作流。
- 你想把某个模板或工作流分享给别人，让对方导入后再本地试跑。

不适合导出：

- 模板还没有跑通。
- 任务强依赖本机账号、时间、地理位置或临时状态。
- 任务里有付款、下单、发消息等高风险动作，且没有在目标设备上测试。

## 导出

导出入口在编辑页：

- 任务模板：打开已保存的模板详情页，点击 **导出模板**。
- 工作流：打开已保存的工作流详情页，点击 **导出工作流**。

新建对象还没有保存时，导出会提示“请先保存后再导出”。

导出成功后，前端会立即弹出结果提示，并显示保存路径。优先保存到：

```text
Downloads/LXB/Tasks/lxb-template-portable-<时间戳>.json
Downloads/LXB/Tasks/lxb-workflow-portable-<时间戳>.json
```

如果系统下载目录不可写，App 会使用自己的外部文件目录作为兜底路径，提示里会显示最终位置。

## 导出文件包含什么？

模板便携包包含：

- 模板名称和描述。
- 目标应用包名。
- 用户操作文档。
- 是否启用旧的 TASK_DECOMPOSE 阶段。
- 路线模式。
- 已保存的模板路线。
- “回放结束后直接结束任务”等路线行为配置。

工作流便携包包含：

- 工作流名称和描述。
- 步骤顺序。
- 每个步骤引用的模板。
- 工作流所需模板的嵌入副本。
- 嵌入模板的路线。

## 不会导出的内容

为了避免把原设备的本机运行状态带到新设备，便携包不会携带：

- 原对象的本地身份。
- 历史运行记录。
- 历史运行时间。
- 工作流触发器配置。
- 触发器启用状态。

导入后，工作流默认触发方式为空、触发器关闭。需要定时或通知触发时，请在工作流编辑页重新配置。

## 导入

导入入口在任务页，按钮名是 **导入便携包**。选择 JSON 文件后，AutoLXB 会自动识别模板便携包、工作流便携包以及兼容的旧格式文件。

导入成功或失败都会立即弹出提示：

- 导入模板成功后，会打开导入模板的编辑页。
- 导入工作流成功后，会打开导入工作流的编辑页。
- 导入失败时，会提示失败原因。

导入时会生成新的本地 ID。允许重名，对象只用 ID 区分。导入过程采用整体事务：模板、工作流或路线任意一部分失败，整次导入都会回滚。

## 旧格式兼容

导入器会识别旧格式任务 JSON / 旧格式路线 JSON，并迁移成当前的模板或工作流结构：

- 旧定时类文件会迁移成一个模板和一个工作流。
- 旧通知触发类文件会迁移成一个模板和一个工作流。
- 纯路线文件会迁移成模板路线。

迁移后仍然需要打开编辑页检查目标应用、任务描述、路线和触发方式。

## 跨设备路线适配

有些路线步骤可以直接在新设备上复用，例如有明确文字或描述的按钮。也有一些步骤在原设备上只能靠坐标或图标位置完成，这类步骤导入后需要在新设备上重新适配。

首次运行导入对象时，AutoLXB 会尝试根据语义说明和当前界面重新找到目标控件，并把适配结果保存到新设备本地。

!!! warning "导入后一定要试跑"
    不同设备的屏幕尺寸、系统字体、App 版本和登录状态都可能不同。导入后建议先手动立即触发一次低风险试跑，确认路线和任务行为都符合预期，再启用定时或通知触发。

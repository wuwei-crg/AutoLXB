# 工作流

工作流用于把一个或多个任务模板按顺序编排起来，并配置一个可选触发方式。工作流本身是静态编排，触发方式是工作流的属性。

当前支持的触发方式：

| 触发方式 | 说明 | 典型场景 |
| --- | --- | --- |
| 无 | 只手动运行。 | 临时组合几个模板，按需点击 **立即触发**。 |
| 定时触发 | 在指定日期、时间和重复规则下自动运行工作流。 | 每天上午运行签到和查询。 |
| 通知触发 | 收到匹配通知后自动运行工作流。 | 收到特定消息后进入 App 处理。 |

保存工作流不会自动启用触发器。工作流列表上的开关只控制触发器启用状态；点击列表里的 **立即触发** 会直接运行工作流，不受触发条件限制。

## 创建工作流

入口：`任务页 -> 工作流 -> 新建`。

基本流程：

1. 先在 **任务模板** 中创建一个或多个模板，写好任务描述、目标应用和用户操作文档。
2. 在 **工作流** 中新建工作流，填写工作流名称。
3. 点击 **添加**，从弹出的模板选择页里选择已有模板加入步骤列表。
4. 根据需要调整步骤顺序。
5. 在 **触发方式** 中选择无、定时触发或通知触发。
6. 保存工作流。
7. 回到工作流列表，根据需要打开触发开关，或点击 **立即触发** 手动运行。

## 基本字段

| 前端字段 / 区域 | 说明 | 是否必填 | 示例 |
| --- | --- | --- | --- |
| 工作流名称 | 用来在工作流列表中识别这组编排。 | 是 | 每日签到 |
| 步骤 | 当前工作流要依次运行的模板列表。首版按顺序执行，不支持嵌套工作流。 | 至少一个 | 瑞幸签到、订单查询 |
| 添加 | 打开模板选择页，从已有任务模板中选择一个加入工作流。 | - | - |
| 触发方式 | 选择这个工作流如何自动触发。 | 是 | 无 / 定时触发 / 通知触发 |
| 触发开关 | 位于工作流列表。只控制当前工作流的触发器是否启用。 | - | 开 |
| 立即触发 | 位于工作流列表。跳过触发条件，立即提交一次工作流运行。 | - | - |

## 定时触发字段

选择 **定时触发** 后，会出现日期、时间和重复设置。

| 前端字段 | 说明 | 是否必填 | 示例 |
| --- | --- | --- | --- |
| 运行时间 | 当前选择的触发时间预览，由日期和时间控件生成。 | 是 | 2026-05-06 11:00 |
| 选择日期 | 打开日期选择器，设置首次或单次执行日期。 | 是 | 2026 年 5 月 6 日 |
| 选择时间 | 打开时间选择器，设置触发时间，使用 24 小时制。 | 是 | 11:00 |
| 重复 | 选择单次、每天或每周。 | 是 | 每天 |
| 已选日期 | 选择每周重复时显示，用于选择周一到周日哪些天执行。 | 每周时必选至少一天 | 周一、周三、周五 |

触发时间已经过去时，先选一个新的时间再保存，避免触发器配置不可用。

## 通知触发字段

选择 **通知触发** 后，会出现通知匹配设置。

| 前端字段 | 说明 | 是否必填 | 示例 |
| --- | --- | --- | --- |
| 选择应用 | 选择要监听哪个 App 的通知。当前界面会从本地已安装应用快照中搜索选择。 | 是 | 微信（com.tencent.mm） |
| 标题匹配 | 匹配通知标题。留空表示不限制标题。 | 否 | WuWei |
| 正文匹配 | 匹配通知正文。留空表示不限制正文。 | 否 | 在吗 |
| 触发间隔（秒） | 同一工作流两次通知触发之间的最短间隔，避免短时间内重复执行。 | 是 | 60 |
| 触发时段开始（HH:mm，可选） | 工作流只在某个时段内响应通知时填写开始时间。 | 否 | 12:00 |
| 触发时段结束（HH:mm，可选） | 工作流只在某个时段内响应通知时填写结束时间。 | 否 | 13:00 |
| 启用 LLM 条件判断 | 开启后，模型会在普通匹配通过后再判断通知是否符合自然语言条件。 | 是 | 开 / 关 |
| LLM 条件 | 给模型看的判断标准。只有开启 LLM 条件判断后才需要填写。 | 否 | 消息来自 WuWei，且是在询问我是否有空 |

!!! note "开始或结束留空表示任意时段"
    前端提示“开始或结束留空时，表示任意时段都可触发”。如果不需要限制触发时间，两个时段字段都可以留空。

## 模板里配置什么？

工作流步骤只引用任务模板。下面这些内容都在模板详情页维护：

- 任务描述。
- 目标应用。点击 **选择应用**，从本地已安装应用快照里搜索并选择。
- 用户操作文档。
- 是否启用旧的 TASK_DECOMPOSE 阶段。默认关闭，避免单模板任务被误拆分。
- 路线开关和任务路线编辑。

标题匹配和正文匹配都是 **包含匹配**。如果标题和正文都填写，则两者都需要匹配成功；如果某一项留空，就表示这一项不限制。

## LLM 条件什么时候用？

普通关键词适合明确、稳定的规则，例如“标题包含某个人名”“正文包含某个订单号”。

如果规则比较模糊，可以启用 LLM 条件判断，例如：

```text
这条通知是在询问我是否有空，或者要求我尽快回复。
```

LLM 条件会在普通包名、标题、正文匹配通过后再执行。它可以提高判断灵活性，但也会增加模型调用成本和延迟。

## 导入后的触发方式

导入工作流便携包时不会导入触发器配置。导入成功后会打开导入对象的编辑页，工作流默认触发方式为空、触发器关闭。需要自动触发时，请重新选择定时触发或通知触发并保存，再回到列表启用触发开关。
